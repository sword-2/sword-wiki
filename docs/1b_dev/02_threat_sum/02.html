Threat Summary

Threat [C/C++ and memory safety]

Caesar (Government)

An information sheet on Software Memory Safety once appeared in google search page. Interest occurred in the content because there are many programming languages that can exceed the memory capacity of some people. Anything to limit the quantity may help simplify. Of course, typical market operation may encourage a bunch of languages. However, the info sheet sheet was summarized as preferring certain computer programming languages and migrating away from C/C++.

Application

One of the SWORD engines was already in Java, a language the information sheet encouraged. This page gave attention to the Swift language for personal reasons of 1) it was a relatively newer language in the sheet's list and 2) swift advertised backward compatibility to C / C++. An individual who came out of C / C++ and hadn't yet learned Java might investigate the newest language, swift in this case.

Perceived challenges

Would using swift be an early plan that was relatively subjective, involved numerous assumptions, and was overly optimistic? This was to be determined. If a challenge was perceived and related to something about Caesar, it was listed below. Perceived challenges more related to the private sector or academics were being moved to another web page on agreements.
