Threat [C/C++ and memory safety]

Caesar (Government)

An information sheet on Software Memory Safety once appeared in a web browser default page and was read. Interest occurred because there are many programming languages that can exceed the memory capacity of some people. Anything to limit the quantity may help simplify. Of course, typical market operation may be encouraged to producing as many products as possible thus being overwhelmed could be more normal than simplifying. The info sheet sheet was summarized as preferring certain computer programming languages and migrating away from C/C++.

Application

One of the SWORD engines was already in Java, a language the information sheet encouraged. This page gave attention to the Swift language for personal reasons of 1) it was a relatively newer language in the sheet's list and 2) swift advertised backward compatibility to C / C++. An individual who came out of C / C++ and hadn't yet learned Java might investigate the newest language, swift in this case.

Perceived challenges

Would using swift be an early plan that was relatively subjective, involved numerous assumptions, and was overly optimistic? This was to be determined.
